# Execution Documents

## Zweck dieses Ordners

Dieser Ordner enthält direkt umsetzbare Arbeitsaufträge für Codex.

Jeder Auftrag steht in einer eigenen Datei, damit er:

- klar referenzierbar ist
- unabhängig überarbeitet werden kann
- ohne Vermischung mit anderen Aufträgen an Codex übergeben werden kann
- einen sauberen Status und eine klare Versionierung behalten kann

## Grundregeln

Jeder Auftrag muss:

- einen klaren Scope haben
- mit dem MVP von PushWrite konsistent sein
- ohne Interpretationsspielraum formuliert sein
- ein überprüfbares Ergebnis verlangen
- Nicht-Ziele explizit benennen
- Akzeptanzkriterien enthalten

## Nicht Zweck dieses Ordners

Dieser Ordner ist:

- kein Ideenspeicher
- kein allgemeines Backlog
- keine Architektur-Dokumentation
- keine Produktbeschreibung

Produkt-, Architektur- und Risikoentscheidungen werden in den jeweils dafür vorgesehenen Dokumenten gepflegt.

## Arbeitsregel

Ein Auftrag wird nur dann angelegt oder geändert, wenn er:

- entscheidungsreif ist
- in sich konsistent ist
- an den aktuellen MVP-Scope gebunden bleibt
- direkt an Codex übergeben werden kann
