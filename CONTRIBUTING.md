# Contributing to PushWrite

Contributions are welcome if they improve the current MVP direction for PushWrite.

## Working Rules

- Keep changes aligned with the macOS MVP.
- Prefer simple, maintainable solutions over added abstraction.
- Treat Whisper as the fixed local transcription engine.
- Do not introduce Windows, Linux, or mobile structure or implementation.
- Do not add file transcription or unrelated feature scope.

## Before Opening a Pull Request

- Check that the change improves the core voice-input workflow.
- Keep the change focused and small enough to review easily.
- Update documentation when scope, structure, or behavior changes.
- Open an issue first for non-trivial scope changes.

## Pull Request Expectations

- Explain what changed and why it is needed for the MVP.
- Avoid unrelated refactoring.
- Avoid premature architecture work.
- Follow the existing repository structure and templates.
